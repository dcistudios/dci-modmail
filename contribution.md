## 📜 Code of Conduct

By participating in this project, you agree to uphold a respectful and welcoming environment. We expect all contributors to:

- Be respectful and constructive in all communication
- Accept feedback gracefully and professionally
- Focus on what is best for the project and community
- Avoid harassment, discrimination, or harmful behavior of any kind

Violations may result in your contributions being rejected or being banned from the project.

## 🚀 Getting Started

Before contributing, make sure you have the following installed:

| Requirement | Version |
|---|---|
| Node.js | 18 or higher |
| npm | 8 or higher |
| Git | Latest stable |

# Fork the repository, then clone your fork
```git clone https://github.com/dcistudios/dci-modmail.git```

# Navigate into the project
```cd dci-modmail```

# Install dependencies
```npm install```

# Copy and configure your environment
```cp .env.example .env```

```cp config.yml.example config.yml```

🛠 How to Contribute
	1.	Fork the repository
	2.	Create a new branch from dev (never directly from main)
	3.	Make your changes following the code style guidelines
	4.	Test your changes thoroughly before submitting
	5.	Open a pull request into the dev branch
⚠️ Pull requests targeting main directly will be closed without review.
---
🐛 Reporting Bugs
Found a bug? Please open a GitHub Issue using the bug report template and include:
	∙	A clear and descriptive title
	∙	Steps to reproduce the issue
	∙	Expected behavior vs. actual behavior
	∙	Your environment (Node.js version, OS, Discord.js version)
	∙	Any relevant error logs or screenshots
Please search existing issues before opening a new one to avoid duplicates.
---
💡 Suggesting Features
Have an idea? Open a GitHub Issue using the feature request template and include:
	∙	A clear description of the feature
	∙	The problem it solves or the value it adds
	∙	Any examples of how it would work
	∙	Whether you’d be willing to implement it yourself
Feature requests will be reviewed by the DCI Studios team. Not all requests will be accepted, but all are appreciated.
---
🎨 Code Style
Please follow these conventions when writing code for this project:
	∙	Use 2 spaces for indentation (no tabs)
	∙	Use single quotes for strings
	∙	Always use const or let — never var
	∙	Add comments for any non-obvious logic
	∙	Keep functions small and focused
	∙	Follow existing file and folder naming conventions
	∙	Avoid committing unused variables or debug logs
